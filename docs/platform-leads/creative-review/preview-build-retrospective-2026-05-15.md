# Preview Build Creative Review Retrospective - 2026-05-15

Purpose: run the new three-archetype creative review process against previously built DirectStay Preview Build work and compare the likely result against what was originally built.

No owner-facing links were created or shared. This is an internal process audit based on existing Preview Build evidence, prior QA notes, source/design briefs, and the 2026-05-15 Sarasota three-variant test.

## Method

For each prior property, score the previous result against the new process criteria:

- visual distinctiveness;
- property/source fit;
- layout hierarchy;
- copy specificity;
- trust/conversion usefulness.

Then identify the archetype the new process would have forced before implementation and whether it would likely improve the result.

## Findings

1. The process should be mandatory before implementation, not only after review. Circle Home proves why: the missing calendar, price, and local-guide modules were obvious only after Jaimal reacted, but the new process would have forced them upfront.

2. The process is most valuable on thin-source or risky-image properties. Sarasota and Savannah benefit heavily because the process separates "make it pretty" from "do not overclaim with unsafe imagery."

3. The old workflow got safer over time but still converged visually. Asheville, Savannah, French Escape, and Sarasota all improved through gates, but the renderer and process still nudged toward similar section/card rhythms. The three-archetype pass is the antidote.

4. The winner should not be adopted raw. In every case, the best production base is usually the most useful/trustworthy variant, while the visual character should be borrowed from the more expressive losing variant.

5. Villa La Percha remains the strongest control, but the new process would still improve it by pushing the preview to compete less with the real site and more clearly demonstrate a reusable DirectStay preview pattern.

## Process Change

Add this requirement to the Preview Build creative loop:

> For every Preview Build, generate or at least explicitly storyboard three materially different archetypes before selecting a final direction. Score each against source fit, layout hierarchy, copy specificity, visual distinctiveness, and trust/conversion usefulness. The final selected direction must include a "borrowed from losing variants" note so the safest version does not become visually bland.

## Recommendation

Use the new process on every future Preview Build and on every old benchmark before citing it as a quality example.

Priority order for rework:

1. Circle Home - highest learning value and weakest old result.
2. Sarasota - convert the planner-first winner into the actual fixture direction when safe imagery exists.
3. Savannah and Asheville - light polish only; their reworked versions are solid, but the process can improve visual distinctiveness.
4. French Escape - keep as a strong lake-house benchmark; add a warmer first-screen emotional layer.
5. Villa La Percha control - update only when needed for sales/demo parity.
