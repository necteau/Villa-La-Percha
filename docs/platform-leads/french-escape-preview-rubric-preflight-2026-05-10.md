## Required fixes/checks before any preview is promoted

1. Render a non-shared internal preview using the content plan and DESIGN brief.
2. Keep date, inquiry, calendar, and price-comparison modules read-only/mock.
3. Produce a page-order image inventory after render; reject promotional graphics, text overlays, OTA badges, ratings, reviews, and duplicate/near-duplicate section art.
4. Run desktop and mobile visual QA.
5. Complete a final rubric review against the rendered URL, including mobile composition and technical polish.
6. Prepare an owner-share note with source basis, draft/noindex/public-obscure/non-functional status, correction path, and removal path before asking Jaimal to approve any external share.

## Recurring weakness to feed back into the playbook

The French Escape packet is safe and property-specific, but the current workflow still allows a gap between a strong planning packet and a reviewable rendered preview. The next playbook/admin-generation improvement should require a **pre-render rubric preflight** and a **post-render promotion gate** so planning quality is not mistaken for owner-share readiness.
